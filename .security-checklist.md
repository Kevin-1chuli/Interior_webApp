# Security Checklist

## ✅ Completed Security Measures

### Environment Variables
- ✅ All `.env` files are in `.gitignore`
- ✅ `backend/.env` is NOT tracked by git
- ✅ All documentation uses placeholders for sensitive values
- ✅ No hardcoded secrets in code files

### Files to NEVER Commit
- `backend/.env` - Production database, JWT, Cloudinary credentials
- `.env.local` - Local development secrets
- Any file containing real DATABASE_URL, JWT_SECRET, or API keys

### Safe Files (Examples Only)
- `backend/.env.example` - Template with placeholder values
- `.env.example` - Frontend template with placeholders

## 🔐 Where Secrets Should Be

### Development (Local)
```
backend/.env (gitignored)
.env.local (gitignored)
```

### Production
```
Railway → Variables tab (backend secrets)
Vercel → Environment Variables (NEXT_PUBLIC_API_URL only)
Neon Dashboard → Connection Details (DATABASE_URL)
Cloudinary Dashboard → Access Keys
```

## ⚠️ If Secrets Were Exposed

If you accidentally committed secrets to git:

### 1. Rotate All Credentials Immediately
- **Neon**: Reset database password in Neon dashboard
- **JWT_SECRET**: Generate new secret: `openssl rand -base64 64`
- **Cloudinary**: Reset API keys in Cloudinary dashboard

### 2. Remove from Git History
```bash
# Remove sensitive file from all commits
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch backend/.env" \
  --prune-empty --tag-name-filter cat -- --all

# Force push (WARNING: rewrites history)
git push origin --force --all
```

### 3. Update All Deployments
- Update Railway environment variables
- Redeploy backend
- Update local `.env` files

## 🛡️ Best Practices

1. **Never print secrets** in logs or console
2. **Never include secrets** in error messages
3. **Use environment variables** for all sensitive data
4. **Rotate credentials** regularly (every 90 days)
5. **Use different secrets** for development vs production
6. **Review commits** before pushing to ensure no secrets included

## 📋 Pre-Commit Checklist

Before every `git commit`:
- [ ] Check no `.env` files are staged: `git status`
- [ ] Verify no secrets in code: `git diff --cached`
- [ ] Ensure only placeholder values in documentation
- [ ] Run: `git ls-files backend/.env` should return empty

## 🔍 Audit Commands

Check for exposed secrets:
```bash
# Check if .env is tracked
git ls-files backend/.env

# Should return nothing (good)

# Search for potential secrets
git grep -i "database_url" -- '*.md'
git grep -i "jwt_secret" -- '*.md'
git grep -i "cloudinary" -- '*.md'

# All should show only placeholder values
```
